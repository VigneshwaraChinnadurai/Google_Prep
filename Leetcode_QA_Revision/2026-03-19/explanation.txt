Title: Count Submatrices With Equal Frequency of X and Y
URL: https://leetcode.com/problems/count-submatrices-with-equal-frequency-of-x-and-y/

LLM Explanation:
The problem asks for the number of submatrices that meet three criteria: they must contain `grid[0][0]`, have an equal frequency of 'X' and 'Y', and include at least one 'X'.

The first condition, containing `grid[0][0]`, is key. It implies that any valid submatrix must start at the top-left corner `(0,0)`. This simplifies the problem immensely, as we only need to consider submatrices defined by their bottom-right corner, `(r, c)`.

The task reduces to iterating through all possible bottom-right corners `(r, c)` and checking if the submatrix from `(0,0)` to `(r,c)` satisfies the other two conditions. To do this efficiently, we use a 2D prefix sum approach to count the occurrences of 'X' and 'Y'.

A standard 2D prefix sum would use O(rows * cols) space. However, we can optimize this to O(cols) space. The insight is that to calculate the prefix sums for the current row `r`, we only need the prefix sum values from the immediately preceding row `r-1`.

The optimized algorithm is as follows:
1.  Initialize two arrays, `ps_x` and `ps_y`, of size `cols` to all zeros. These will store the column-wise prefix sums up to the previous row.
2.  Iterate through the grid row by row, from `r = 0` to `rows-1`.
3.  For each row, maintain `row_x_sum` and `row_y_sum` to track the running sum of 'X's and 'Y's within the current row.
4.  Iterate through each column `c` from `0` to `cols-1`.
    a.  Update `row_x_sum` and `row_y_sum` based on the character `grid[r][c]`.
    b.  The total count of 'X's for the submatrix ending at `(r,c)`, `total_x`, is the sum of 'X's in the submatrix above it (ending at `(r-1, c)`, which is `ps_x[c]`) plus the sum of 'X's in the current row up to column `c` (`row_x_sum`). The same logic applies to `total_y`.
    c.  Check if the conditions are met: `total_x > 0` and `total_x == total_y`. If so, increment the result counter.
    d.  Update `ps_x[c]` and `ps_y[c]` with `total_x` and `total_y`. This prepares the arrays for the calculation of the next row.

This method processes each cell of the grid once.

-   **Time Complexity:** O(rows * cols), as we iterate through each cell of the grid once.
-   **Space Complexity:** O(cols), for the two arrays used to store the prefix sums of the previous row.

This approach correctly handles edge cases like empty grids or grids without any 'X's.
