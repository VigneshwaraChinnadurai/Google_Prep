Title: Rotate Image
URL: https://leetcode.com/problems/rotate-image/

LLM Explanation:
The problem requires rotating an n x n matrix 90 degrees clockwise in-place. A clever and efficient way to achieve this is by combining two simpler transformations: transposing the matrix and then reversing each row.

### Approach: Transpose and Reverse

1.  **Transpose the Matrix**: The first step is to transpose the matrix. A transpose operation flips the matrix over its main diagonal, which means `matrix[i][j]` is swapped with `matrix[j][i]`. To do this in-place, we can iterate through the matrix with nested loops. The outer loop runs from `i = 0` to `n-1`, and the inner loop from `j = i + 1` to `n-1`. This ensures we only swap the elements in the upper triangle of the matrix with their corresponding elements in the lower triangle, avoiding redundant swaps.

    For example, `[[1,2],[3,4]]` becomes `[[1,3],[2,4]]` after transposing.

2.  **Reverse Each Row**: After transposing, the second step is to reverse the elements in each row. Iterating through each row and reversing it will complete the 90-degree clockwise rotation.

    Continuing the example, `[[1,3],[2,4]]` becomes `[[3,1],[4,2]]` after reversing each row, which is the correct 90-degree clockwise rotation of the original matrix.

This two-step process correctly repositions every element to its rotated destination. The entire operation is performed on the input matrix itself, satisfying the in-place requirement.

### Complexity Analysis

-   **Time Complexity**: O(N^2), where N is the number of rows (or columns) in the matrix.
    -   The transpose step involves visiting each element in the upper triangle once, which is roughly N^2 / 2 operations.
    -   The reverse step involves visiting each of the N rows and reversing them. Reversing a row of length N takes O(N) time. So, this step takes N * O(N) = O(N^2).
    -   The total time complexity is O(N^2) + O(N^2) = O(N^2).

-   **Space Complexity**: O(1). The rotation is performed in-place. We only use a few variables for loops and swaps, which does not depend on the size of the matrix.

### Edge Cases

-   **1x1 Matrix**: The loops for transpose will not run. The loop for reversing the single-element row also does nothing. The matrix remains unchanged, which is correct.
-   **Empty Matrix**: The problem constraints state `1 <= n <= 20`, so an empty matrix is not a valid input. However, the code would handle it gracefully (`n=0`, loops don't execute).
