Title: Minimum Distance to the Target Element
URL: https://leetcode.com/problems/minimum-distance-to-the-target-element/

LLM Explanation:
The problem asks us to find the minimum absolute difference between a given `start` index and an index `i` where `nums[i]` is equal to `target`.

The most straightforward approach is to iterate through the entire `nums` array and check every element.

1.  Initialize a variable, `min_distance`, to a very large value (e.g., infinity). This variable will store the minimum distance found so far.
2.  Iterate through the `nums` array using `enumerate` to get both the index `i` and the value `num` at that index.
3.  For each element, check if `num` is equal to the `target`.
4.  If it is, calculate the distance to the `start` index: `distance = abs(i - start)`.
5.  Compare this `distance` with the current `min_distance`. If the new distance is smaller, update `min_distance` to this new value.
6.  After checking all elements in the array, `min_distance` will hold the minimum possible distance. The problem guarantees that the `target` exists in `nums`, so we are sure to find at least one match.

This approach ensures we check all possible occurrences of the `target` and find the one closest to the `start` index.

A more concise, "Pythonic" way to write this is using a generator expression with the `min()` function, which achieves the same result:
`return min(abs(i - start) for i, num in enumerate(nums) if num == target)`

**Complexity Analysis:**
-   **Time Complexity:** O(N), where N is the number of elements in `nums`. We must scan the entire array in the worst case to find all occurrences of the target.
-   **Space Complexity:** O(1), as we only use a few variables to store the minimum distance and loop counters, regardless of the input size.
