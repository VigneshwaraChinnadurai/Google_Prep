Title: Maximum Path Score in a Grid
URL: https://leetcode.com/problems/maximum-path-score-in-a-grid/

LLM Explanation:
This problem asks for the maximum score of a path from the top-left to the bottom-right of a grid, with a constraint on the total cost of the path. Since we can only move right or down, this structure is ideal for a dynamic programming approach.

The state of our DP needs to capture not only the position in the grid but also the accumulated cost, as this is crucial for making future decisions. Let's define a DP state `dp[i][j][c]` as the maximum score to reach cell `(i, j)` with an exact total cost of `c`.

The transition to compute `dp[i][j][c]` would involve looking at the maximum scores from the cell above, `(i-1, j)`, and the cell to the left, `(i, j-1)`.
Let `cell_score` and `cell_cost` be the score and cost of the current cell `(i, j)`.
A path reaching `(i, j)` with cost `c` must have come from a previous cell with cost `c - cell_cost`.
So, the recurrence relation is:
`dp[i][j][c] = max(dp[i-1][j][c - cell_cost], dp[i][j-1][c - cell_cost]) + cell_score`

A DP table of size `m x n x (k+1)` would work, but we can optimize the space. Notice that to compute the values for the current row `i`, we only need the values from the previous row `i-1` and the current row `i` itself. This allows us to reduce the space complexity from `O(m * n * k)` to `O(n * k)`.

The implementation uses a 2D array `dp` of size `n x (k+1)`. `dp[j][c]` stores the maximum score to reach column `j` in the current row being processed, with a total cost of `c`. We iterate through the grid row by row. When computing the DP values for cell `(i, j)`, `dp[j]` still holds the information from `(i-1, j)`, while `dp[j-1]` has already been updated with the information for `(i, j-1)`.

The algorithm proceeds as follows:
1. Initialize an `n x (k+1)` DP table, `dp`, with -1 to represent unreachable states.
2. Set the base case: `dp[0][0] = 0`, as we start at `(0,0)` with 0 score and 0 cost (since `grid[0][0]` is always 0).
3. Iterate through each cell `(i, j)` of the grid.
4. For each cell, calculate the possible maximum scores by considering paths from above `(i-1, j)` and from the left `(i, j-1)`.
5. A temporary array `current_cell_dp` is used to store the new values for `(i, j)` to avoid overwriting the data from `(i-1, j)` which is needed for the next cell in the same row.
6. After computing the new values for `(i, j)`, update `dp[j]`.
7. After iterating through the entire grid, the last row of our DP table, `dp[n-1]`, contains the maximum scores to reach the bottom-right cell `(m-1, n-1)` for every possible cost from 0 to `k`.
8. The final answer is the maximum value in `dp[n-1]`. If no path is valid, all values will be -1, and the maximum will also be -1.

**Complexity Analysis:**
- **Time Complexity:** `O(m * n * k)`, due to the three nested loops iterating through rows, columns, and costs.
- **Space Complexity:** `O(n * k)`, for the space-optimized DP table.
