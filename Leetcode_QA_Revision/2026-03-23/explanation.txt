Title: Maximum Non Negative Product in a Matrix
URL: https://leetcode.com/problems/maximum-non-negative-product-in-a-matrix/

LLM Explanation:
### Method: Dynamic Programming

This problem asks for the maximum non-negative product of a path in a grid, which is a classic dynamic programming (DP) problem. The key challenge is handling the negative numbers in the grid. A simple DP approach of only tracking the maximum product at each cell is insufficient because a path with a very small (large magnitude negative) product could become the maximum positive product in the next step if multiplied by another negative number.

To address this, we need to maintain two DP tables for each cell `(i, j)`:
1.  `dp_max[i][j]`: The maximum product of a path from `(0, 0)` to `(i, j)`.
2.  `dp_min[i][j]`: The minimum product of a path from `(0, 0)` to `(i, j)`.

#### DP State and Recurrence Relation

A cell `(i, j)` can only be reached from the cell above, `(i-1, j)`, or the cell to the left, `(i, j-1)`. The new maximum and minimum products at `(i, j)` depend on the sign of `grid[i][j]`:

-   **If `grid[i][j]` is non-negative:**
    -   To get the new maximum product, we multiply `grid[i][j]` by the largest product from the preceding cells: `max(dp_max[i-1][j], dp_max[i][j-1]) * grid[i][j]`.
    -   To get the new minimum product, we multiply `grid[i][j]` by the smallest product from the preceding cells: `min(dp_min[i-1][j], dp_min[i][j-1]) * grid[i][j]`.

-   **If `grid[i][j]` is negative:**
    -   The roles are reversed. To get the new maximum product, we must multiply `grid[i][j]` by the most negative product from the preceding cells: `min(dp_min[i-1][j], dp_min[i][j-1]) * grid[i][j]`.
    -   To get the new minimum product, we multiply by the most positive product: `max(dp_max[i-1][j], dp_max[i][j-1]) * grid[i][j]`.

#### Algorithm Steps

1.  Initialize two `m x n` DP tables, `dp_max` and `dp_min`.
2.  **Base Case:** For the starting cell `(0, 0)`, `dp_max[0][0]` and `dp_min[0][0]` are both equal to `grid[0][0]`.
3.  **Initialization:** Fill the first row and first column of both tables. For these cells, there is only one possible path to reach them.
4.  **Iteration:** Iterate through the rest of the grid from `(1, 1)` to `(m-1, n-1)`, applying the recurrence relations described above to fill `dp_max[i][j]` and `dp_min[i][j]`.
5.  **Result:** The maximum possible product for any path ending at the bottom-right corner is `dp_max[m-1][n-1]`.
    -   If this value is non-negative, return it modulo `10^9 + 7`.
    -   If it's negative, it means no path can yield a non-negative product, so return `-1`.

#### Complexity

-   **Time Complexity:** `O(m * n)`, as we iterate through each cell of the grid once.
-   **Space Complexity:** `O(m * n)` for the two DP tables. This can be optimized to `O(n)` since each cell's calculation only depends on the previous row and the current row. However, given the small constraints (`m, n <= 15`), the `O(m * n)` space solution is perfectly acceptable and more straightforward to implement.

LLM Testcase Validation:
- Testcase 1: `grid = [[-1,-2,-3],[-2,-3,-3],[-3,-3,-2]]`
  - The path from (0,0) to (2,2) has a length of 5 (m+n-1). An odd number of negative values will result in a negative product. All numbers are negative, so any path will have a negative product. The maximum product at `dp_max[2][2]` will be negative. The function correctly returns -1.
- Testcase 2: `grid = [[1,-2,1],[1,-2,1],[3,-4,1]]`
  - The DP table for maximum products `dp_max` will be populated. For example, `dp_max[1][1]` is `max(dp_max[0][1], dp_max[1][0])` if `grid[1][1]` were positive, but since `grid[1][1] = -2` is negative, it becomes `min(dp_min[0][1], dp_min[1][0]) * -2 = min(-2, 1) * -2 = 4`. Following this logic, `dp_max[2][2]` will be 8. Since 8 is non-negative, the function returns `8 % (10^9 + 7)`, which is 8.
- Testcase 3: `grid = [[1,3],[0,-4]]`
  - Path 1: 1 -> 3 -> -4, product = -12.
  - Path 2: 1 -> 0 -> -4, product = 0.
  - The maximum non-negative product is 0. The DP calculation will find `dp_max[1][1] = 0`. The function returns `0 % (10^9 + 7)`, which is 0.
