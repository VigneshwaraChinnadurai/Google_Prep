Title: Find Minimum in Rotated Sorted Array
URL: https://leetcode.com/problems/find-minimum-in-rotated-sorted-array/

LLM Explanation:
The problem requires finding the minimum element in a rotated sorted array in O(log n) time. This time complexity strongly suggests a binary search algorithm.

A key property of a rotated sorted array (with unique elements) is that it consists of two sorted subarrays. For example, `[4,5,6,7,0,1,2]` is composed of `[4,5,6,7]` and `[0,1,2]`. The minimum element is always the first element of the second subarray, which acts as the pivot point of the rotation.

The binary search strategy is to efficiently discard half of the search space in each iteration by determining which half contains this pivot point. We can do this by comparing the middle element, `nums[mid]`, with one of the boundary elements, such as `nums[right]`.

The algorithm proceeds as follows:
1.  Initialize two pointers, `left = 0` and `right = len(nums) - 1`.
2.  While `left < right`, we are searching in a space with at least two elements:
    a. Calculate the middle index: `mid = left + (right - left) // 2`.
    b. Compare `nums[mid]` with `nums[right]`:
        - If `nums[mid] > nums[right]`: This means the inflection point (the minimum value) must be in the right half of the current search space, i.e., `(mid, right]`. The segment from `left` to `mid` is sorted but contains values larger than the elements in the other rotated part. We can safely discard the left half by setting `left = mid + 1`.
        - If `nums[mid] <= nums[right]`: This means the segment from `mid` to `right` is sorted in ascending order. The minimum element could be `nums[mid]` itself or an element to its left. Thus, we can discard the portion to the right of `mid` by setting `right = mid`.
3.  The loop terminates when `left` and `right` converge to a single index (`left == right`). This index points to the minimum element in the array.

This approach correctly handles all cases, including arrays that are not rotated (or rotated `n` times), and edge cases like single-element arrays.

- **Time Complexity:** O(log n), as the search space is halved in each step of the binary search.
- **Space Complexity:** O(1), as we only use a constant amount of extra space for pointers.

LLM Testcase Validation:
- **Input:** `nums = [3,4,5,1,2]`
  - **Execution:**
    - `left=0, right=4`. `mid=2` (`nums[2]=5`). `5 > nums[4]` (2). `left` becomes `3`.
    - `left=3, right=4`. `mid=3` (`nums[3]=1`). `1 < nums[4]` (2). `right` becomes `3`.
    - Loop terminates as `left == right`.
  - **Output:** `nums[3]` which is `1`. Correct.
