Title: Shortest Distance to Target String in a Circular Array
URL: https://leetcode.com/problems/shortest-distance-to-target-string-in-a-circular-array/

LLM Explanation:
The problem asks for the shortest distance from a `startIndex` to a `target` string in a circular array. This can be viewed as a shortest path problem on a cycle graph.

A highly effective strategy is to search outwards from the `startIndex` in both directions simultaneously. This approach is analogous to a Breadth-First Search (BFS), which naturally finds the shortest path in an unweighted graph. We explore nodes (array indices) at distance 0, then distance 1, then distance 2, and so on. The first time we encounter the `target`, we are guaranteed to have found it via a shortest path.

The algorithm is as follows:
1.  We use a loop that iterates a `dist` variable from 0 upwards. This `dist` represents the number of steps taken from `startIndex`.
2.  In each iteration `dist`, we check two indices:
    - The index `dist` steps clockwise (to the right): `(startIndex + dist) % n`.
    - The index `dist` steps counter-clockwise (to the left): `(startIndex - dist + n) % n`.
    The modulo operator `% n` correctly handles the wrap-around logic of the circular array.
3.  If the `target` string is found at either of these two indices, we can immediately return the current `dist`, as this is the shortest possible distance.
4.  The search for `dist` only needs to go up to `n // 2`, where `n` is the length of the array. This is because the maximum shortest distance between any two points in a circular array of size `n` is `n // 2`. This range of distances is sufficient to check every index in the array.
5.  If the loop completes without finding the `target`, it implies the `target` does not exist in the `words` array. In this scenario, we return -1.

This method is efficient and clean. The time complexity is O(N * L), where N is the number of words and L is the maximum length of a word. In the worst case, we might check all N words, and each check involves a string comparison taking O(L) time. The space complexity is O(1) as we only use a constant amount of extra space.

- Testcase 1:
Input: words = ["hello","i","am","leetcode","hello"], target = "hello", startIndex = 1
n = 5. Loop for dist in range(3) -> 0, 1, 2.
dist = 0: Check index (1+0)%5=1 and (1-0+5)%5=1. words[1] is "i", not "hello".
dist = 1: Check index (1+1)%5=2. words[2] is "am". Check index (1-1+5)%5=0. words[0] is "hello". Match found. Return dist = 1.
Output: 1. Correct.
