Title: Check if Array is Good
URL: https://leetcode.com/problems/check-if-array-is-good/

LLM Explanation:
An array `nums` is defined as "good" if it is a permutation of `base[n]`, where `base[n]` consists of numbers from `1` to `n-1` each occurring once, and the number `n` occurring twice.

The core insight is that the properties of `base[n]` impose strict constraints on any "good" array `nums`.
1.  **Length Constraint:** The length of `base[n]` is `(n-1) + 2 = n+1`. Therefore, if `nums` is a good array, its length must be `n+1`. This allows us to deduce the only possible value for `n`: `n = len(nums) - 1`.
2.  **Content Constraint:** The elements of `nums` must match the frequencies in `base[n]`.

This leads to an efficient algorithm using a frequency map:

1.  First, we determine the candidate `n` from the length of the input array: `n = len(nums) - 1`.
2.  The smallest possible "good" array is `base[1] = [1, 1]`, which has a length of 2. Thus, any array with a length less than 2 (i.e., `n < 1`) cannot be good.
3.  We use a hash map (specifically, `collections.Counter` in Python) to count the occurrences of each number in `nums`.
4.  We then verify if these counts match the definition of `base[n]`:
    - The count of the number `n` must be exactly 2.
    - The count of every number `i` from `1` to `n-1` must be exactly 1.
5.  If all these conditions are met, the array is good. The total number of elements accounted for by these checks is `(n-1) * 1 + 2 = n+1`, which is the length of `nums`. This guarantees that no other extraneous numbers are present, so we don't need to check for them explicitly.

This approach has a time complexity of O(N) because we iterate through the array once to build the counter and then iterate from 1 to `n` (where `n` is N-1), resulting in linear time. The space complexity is O(N) to store the frequency map.

LLM Testcase Validation:
- `nums = [2, 1, 3]` -> `n = 2`. `counts = {1:1, 2:1, 3:1}`. `counts[2]` is 1, not 2. Returns `False`. Correct.
- `nums = [1, 3, 3, 2]` -> `n = 3`. `counts = {1:1, 2:1, 3:2}`. `counts[3]` is 2. `counts[1]` is 1. `counts[2]` is 1. All checks pass. Returns `True`. Correct.
- `nums = [1, 1]` -> `n = 1`. `counts = {1:2}`. `counts[1]` is 2. The loop for `1..n-1` is empty. All checks pass. Returns `True`. Correct.
- `nums = [3, 4, 4, 1, 2, 1]` -> `n = 5`. `counts = {1:2, 2:1, 3:1, 4:2}`. `counts[5]` is 0, not 2. Returns `False`. Correct.
