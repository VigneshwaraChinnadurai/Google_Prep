Title: Closest Equal Element Queries
URL: https://leetcode.com/problems/closest-equal-element-queries/

LLM Explanation:
The problem asks for the minimum circular distance from a given index to another index with the same value. A naive approach of iterating through the array for each query would be too slow, with a time complexity of O(N*Q). A more efficient solution involves preprocessing and binary search.

1.  **Preprocessing:** We first create a hash map (a `defaultdict` in Python) to group all indices by their corresponding value in `nums`. We iterate through `nums` once, and for each value, we append its index to a list in the hash map. Since we iterate in order, each list of indices will be naturally sorted. This step takes O(N) time.

2.  **Query Processing:** For each `query_index`, we perform the following steps:
    a.  Look up the list of indices for `nums[query_index]` in our pre-built map.
    b.  If the list contains only one element, it means the value is unique, so we record -1 as the answer.
    c.  If the list has multiple indices, we need to find the one closest to `query_index`. Since the list is sorted, the closest indices must be the immediate predecessor and successor. We can find the position of `query_index` in this list efficiently using binary search (Python's `bisect.bisect_left`).

3.  **Calculating Circular Distance:** Once we identify the predecessor and successor indices, we calculate the circular distance to each. The distance between two indices `i` and `j` in a circular array of length `n` is `min(|i - j|, n - |i - j|)`. We must also handle the wrap-around case for neighbors: the predecessor of the first element is the last, and the successor of the last element is the first. This is easily managed with Python's negative indexing and the modulo operator.

4.  **Final Answer:** The minimum of the circular distances to the predecessor and successor is the answer for the query.

This approach has a time complexity of O(N + Q * log K), where N is the length of `nums`, Q is the number of queries, and K is the maximum frequency of any number. The space complexity is O(N) for the hash map. This is highly efficient and passes within the given constraints.
