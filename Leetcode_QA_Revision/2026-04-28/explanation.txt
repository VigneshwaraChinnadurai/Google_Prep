Title: Minimum Operations to Make a Uni-Value Grid
URL: https://leetcode.com/problems/minimum-operations-to-make-a-uni-value-grid/

LLM Explanation:
The problem asks whether a valid path exists from the top-left `(0, 0)` to the bottom-right `(m-1, n-1)` of a grid, where movement is constrained by street types in each cell. This can be modeled as a graph traversal problem where grid cells are nodes and valid connections are edges.

A Breadth-First Search (BFS) is an excellent algorithm for determining path existence in an unweighted graph. The core of the solution is to correctly define when a move from one cell to an adjacent one is "valid".

A move from cell `(r, c)` to an adjacent cell `(nr, nc)` is valid if and only if two conditions are met:
1. The street at `(r, c)` has an opening/port pointing towards `(nr, nc)`.
2. The street at `(nr, nc)` has an opening/port pointing back towards `(r, c)`.

The algorithm proceeds as follows:
1.  **Data Structure for Moves**: A dictionary `moves` is created to map each street type (integer 1-6) to a set of allowed move directions, represented as `(dr, dc)` tuples. For example, street type 1 (left-right) maps to `{(0, -1), (0, 1)}`. Using sets allows for efficient O(1) lookups.
2.  **BFS Initialization**: A standard BFS is set up using a queue, initialized with the starting cell `(0, 0)`. A `visited` set is used to keep track of cells that have already been processed to avoid cycles and redundant computations.
3.  **Traversal Logic**:
    - The BFS dequeues a cell `(r, c)`.
    - If `(r, c)` is the destination, a path has been found, and we return `true`.
    - Otherwise, for the current cell's street type, we iterate through its possible moves `(dr, dc)`.
    - For each move, we calculate the neighbor's coordinates `(nr, nc)`.
    - We check if the move is valid:
        - The neighbor must be within the grid boundaries.
        - The neighbor must not have been visited.
        - The crucial connection check: The street at `(nr, nc)` must support the reverse move `(-dr, -dc)`. This ensures the two streets connect properly.
    - If the move is valid, the neighbor is added to the queue and the `visited` set.
4.  **Termination**: If the queue becomes empty and the destination has not been reached, it means no valid path exists, and the function returns `false`.

This BFS approach systematically explores all reachable cells from the start, guaranteeing that if a path exists, it will be found.

**Complexity Analysis**:
-   **Time Complexity**: `O(m * n)`, where `m` and `n` are the grid dimensions. Each cell is enqueued and processed at most once.
-   **Space Complexity**: `O(m * n)` in the worst case, for storing the cells in the `queue` and the `visited` set.

LLM Testcase Validation:
- Testcase 1: `grid = [[2,4,3],[6,5,2]]`
  - Expected output: `true`
  - My code's logic: A valid path exists: (0,0) -> (1,0) -> (1,1) -> (0,1) -> (0,2) -> (1,2). The algorithm will find this path and return `true`.

- Testcase 2: `grid = [[1,2,1],[1,2,1]]`
  - Expected output: `false`
  - My code's logic: The starting cell (0,0) has street type 1 (left-right). It can't connect to the cell (0,1) which has street type 2 (up-down), as type 2 has no left-facing port. The path is blocked at the start. The algorithm returns `false`.

- Testcase 3: `grid = [[1,1,2]]`
  - Expected output: `false`
  - My code's logic: The path can go from (0,0) to (0,1) as both are type 1. However, from (0,1), it cannot connect to (0,2) which is type 2 (up-down) because type 2 has no left-facing port. The path is blocked at (0,1). The algorithm returns `false`.
