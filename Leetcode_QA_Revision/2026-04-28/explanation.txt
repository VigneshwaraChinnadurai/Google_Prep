Title: Minimum Operations to Make a Uni-Value Grid
URL: https://leetcode.com/problems/minimum-operations-to-make-a-uni-value-grid/

LLM Explanation:
The problem asks for the minimum number of operations to make all elements in a grid equal, where an operation consists of adding or subtracting `x` from an element.

First, we must determine if a solution is possible. For any two numbers, `a` and `b`, to be made equal to a common value `v`, `v` must be reachable from both. This means `v = a + k1*x` and `v = b + k2*x` for some integers `k1` and `k2`. This implies `a - b = (k2 - k1)*x`, meaning the difference between `a` and `b` must be a multiple of `x`. This condition must hold for all pairs of elements in the grid. A simpler way to check this is to verify that all elements have the same remainder when divided by `x`. If this condition is not met, it's impossible to make the grid uni-value, and we return -1.

If a solution is possible, we need to find a target value `v` that minimizes the total number of operations. The number of operations to change an element `num` to `v` is `abs(num - v) / x`. The total number of operations is the sum of these individual operation counts over all elements: `sum(abs(num - v) / x)`. To minimize this sum, we need to minimize the numerator, `sum(abs(num - v))`.

This is a classic statistical problem: finding a point `v` that minimizes the sum of absolute differences to a set of numbers. The optimal choice for `v` is the median of the numbers.

The overall algorithm is as follows:
1.  Flatten the 2D grid into a single 1D list of numbers.
2.  Check the feasibility condition: iterate through the list and ensure all numbers have the same remainder when divided by `x`. If not, return -1.
3.  Sort the flattened list to find the median. For a list of size `N`, the median is the element at index `N // 2`.
4.  Calculate the total number of operations. The chosen median is one of the grid elements, so it's a valid target. The total operations is the sum of `abs(num - median) // x` for each number `num` in the list.

The time complexity is dominated by the sorting step, which is `O(N log N)`, where `N = m * n` is the total number of elements in the grid. The space complexity is `O(N)` to store the flattened list.

LLM Testcase Validation:
- Testcase 1: `grid = [[2,4],[6,8]], x = 2`
  - Flattened list: `[2, 4, 6, 8]`
  - All numbers have remainder 0 when divided by 2. Possible.
  - Sorted list: `[2, 4, 6, 8]`
  - Median: `nums[4 // 2] = nums[2] = 6`
  - Operations: `(|2-6|/2) + (|4-6|/2) + (|6-6|/2) + (|8-6|/2) = 2 + 1 + 0 + 1 = 4`.
  - Expected: 4. Correct.

- Testcase 2: `grid = [[1,5],[2,3]], x = 1`
  - Flattened list: `[1, 5, 2, 3]`
  - All numbers have remainder 0 when divided by 1. Possible.
  - Sorted list: `[1, 2, 3, 5]`
  - Median: `nums[4 // 2] = nums[2] = 3`
  - Operations: `(|1-3|/1) + (|2-3|/1) + (|3-3|/1) + (|5-3|/1) = 2 + 1 + 0 + 2 = 5`.
  - Expected: 5. Correct.

- Testcase 3: `grid = [[1,2],[3,4]], x = 2`
  - Flattened list: `[1, 2, 3, 4]`
  - `1 % 2 = 1`, but `2 % 2 = 0`. Remainders are different. Impossible.
  - The code will detect this and return -1.
  - Expected: -1. Correct.
